## De quoi parle-t-on?
— Hey Guillaume! Le sujet du jour est "les tests unitaires". J'espère que tu as du temps pour en discuter mais d'abord, permets-moi de l'introduire.

Le testing unitaire (anglais, _unit testing_) est une partie d'une stratégie plus grosse appelée "test de logiciel" (anglais, _software testing_). L'idée derrière ces mots est de simplement tester les fonctionnalités designées par des analystes et développeurs par des ingénieurs.
Il a pas mal de types de testing, et comme tu vas le remarquer dans l'arbre si-dessous, les types sont divisées en 2 branches:
- les tests fonctionnels – ça signifie que leur objectif cible le code lui-même
- les tests non-fonctionnels – leur but est plus global, c'est du testing de l'application de manière générale ou tous les sujets qui gravitent autour comme la sécurité, les environnements, la documentation, etc.

<Figure data={{src: '/img/software-testing-types.png', alt: 'Types de Software Testing', type: 'image'}} />

Avec Guillaume, nous allons parler de tests unitaires parce que c'est une bonne première étape quand on veut mettre un pied dans le testing de logiciel. Mais n'aie pas peur de regarder aux autres sur le net!
